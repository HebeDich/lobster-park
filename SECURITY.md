# Security Policy

## Supported Versions

当前仓库处于早期开源整理阶段，默认只接受最新 `main` 分支相关的安全问题反馈。

## Reporting a Vulnerability

请不要在公开 Issue 中直接披露安全漏洞细节。

建议方式：

1. 优先使用仓库平台提供的私密漏洞报告通道（如 GitHub Security Advisories）
2. 如果仓库平台未开启私密报告通道，请通过维护者公开提供的私下联系方式提交
3. 标题使用 `Security: <short summary>`
4. 附带影响范围、复现步骤、潜在风险和建议修复方向

## Response Expectations

- 维护者应在 5 个工作日内确认收到报告
- 在问题确认后协调修复与披露时间
- 在修复发布前，不公开 exploit 或敏感细节
