## 1. 项目背景

企业希望基于 OpenClaw 构建统一控制平台，让内部员工无需命令行和复杂部署知识，也能自助创建、配置、运行并维护自己的 OpenClaw 应用实例，从而降低 AI Agent 使用门槛并提升办公效率。

平台暂定名称：**龙虾乐园**。

龙虾乐园不是 OpenClaw 的替代品，而是其上层的**企业控制平面（Control Plane）**，负责：

1. 实例生命周期管理
2. 配置中心与版本发布
3. 节点接入与审批
4. 平台级监控、告警、通知、报表
5. 多租户、RBAC、安全治理、审计
6. 运行规格、版本兼容、隔离策略

## 2. 产品定位与立项结论

### 2.1 产品定位

龙虾乐园是面向企业的 OpenClaw 统一管理平台，目标不是重做 Agent Runtime，而是补齐 OpenClaw 在企业场景中最需要的：

- 多用户治理
- 多实例管理
- 运行时隔离
- 可视化配置
- 版本发布回滚
- 统一监控告警
- 安全与合规控制

### 2.2 立项结论

该项目**可行，且适合作为企业内部试点产品先行落地**。  
平台侧只对运行时做标准化封装，不侵入 OpenClaw 核心业务逻辑，优先通过 Runtime Adapter / Orchestrator 统一编排。

### 2.3 核心架构原则

1. **实例级隔离优先**：V1 不默认采用“多员工共享一个 OpenClaw runtime”模式。
2. **控制平面与运行平面解耦**：平台只维护元数据、权限、发布、监控与编排。
3. **Schema 驱动配置优先**：优先兼容上游配置 schema，减少硬编码表单。
4. **最小权限与默认安全关闭**：公共技能市场默认关闭，高危能力默认不开放。
5. **先内部标准化，再考虑外部商业化**：V1 数据模型按多租户设计，但先服务内部试点。

## 3. 产品目标

### 3.1 业务目标

1. 员工可在 **5~10 分钟**内创建一个可用实例。
2. 大幅降低安装、配置、升级、排障的人工成本。
3. 确保用户之间、实例之间、租户之间的隔离性。
4. 提供可观测、可审计、可治理的平台能力。
5. 为后续商业化版本预留多租户、版本、规格、通知、计费扩展能力。

### 3.2 成功标准（V1）

1. 员工无需命令行即可完成实例创建、基础配置、发布与启动。
2. 同一租户可管理多个用户、多个实例、多个节点。
3. 管理员可查看实例健康、节点状态、基础使用量、告警与审计。
4. 配置变更可校验、可发布、可回滚，失败时不影响当前生效版本。
5. 平台默认禁止用户访问他人实例的配置、密钥、日志、节点、告警。
6. 技能启用受企业白名单和审核机制控制。

### 3.3 用户故事

1. **作为普通员工**，我希望在 5 分钟内创建一个可用实例，以便快速获得 AI 助手能力。
2. **作为普通员工**，我希望看到配置校验的具体错误信息，以便自行修复配置问题。
3. **作为普通员工**，我希望在实例异常时收到通知，并能快速定位是模型、channel 还是节点问题。
4. **作为租户管理员**，我希望看到租户内所有实例的健康概览，以便及时处理异常。
5. **作为租户管理员**，我希望审批节点配对申请，以便控制设备接入。
6. **作为平台管理员**，我希望维护允许使用的 OpenClaw 版本、技能来源和资源规格，以便控制风险和成本。
7. **作为安全审计人员**，我希望查看关键操作审计记录，以便满足合规要求。

## 4. 目标用户与角色

### 4.1 目标用户

当前阶段：企业内部员工与管理人员。

### 4.2 角色定义

1. **平台超级管理员**
   - 管理全局租户、版本 allowlist、资源规格、模板后台、技能白名单、全局监控与审计。

2. **租户管理员**
   - 管理本租户用户、角色、实例、节点审批、告警确认、租户级报表。

3. **普通员工**
   - 创建并管理自己有权限的实例、编辑配置、发布配置、查看健康状态、申请节点接入。

4. **安全/审计角色**
   - 只读查看审计、告警、配置变更记录、技能审核记录，无运行操作权限。

## 7. 核心对象模型

### 7.1 Tenant
- 企业级隔离单元
- 具备自己的成员、角色、实例、节点审批、报表与告警视图

### 7.2 User
- 平台登录主体
- 属于一个租户
- 可同时拥有多个角色与实例权限

### 7.3 Instance
- 龙虾乐园管理的最小运行单元
- 对应一个独立 OpenClaw runtime
- 拥有独立配置、workspace、state、log、secret scope、节点关系和运行绑定

### 7.4 Config Draft / Config Version
- Draft：可编辑、未生效
- Version：可校验、可发布、可回滚的版本快照

### 7.5 Node / Pairing Request
- Node：已接入或历史接入设备
- Pairing Request：节点发起、等待审批的流程记录

### 7.6 Skill Package
- 来源可以是平台内置、企业私有仓库、审核导入的第三方技能
- V1 不允许用户任意连接公共市场

### 7.7 Alert / Notification
- Alert：监控事件
- Notification：告警、任务、审批等事件触发的消息分发结果

## 8. 详细功能需求

## 8.1 登录与身份体系

### 功能说明
1. 平台支持企业统一登录，V1 默认采用 **OIDC SSO**。
2. 如企业内部仅具备 LDAP / AD，优先通过企业身份网关接入，而不是平台直接对接 LDAP。
3. 登录成功后，平台按租户和角色下发权限上下文。

### 验收标准
1. 未登录用户无法访问业务页面。
2. 登录后根据角色看到不同导航和操作按钮。
3. 同一用户在不同租户的权限上下文可被正确识别（若启用多租户）。

## 8.2 租户、用户与 RBAC

### 功能说明
1. 支持租户、用户、角色、权限模型。
2. 权限粒度覆盖租户、用户、实例、配置、节点、技能、审计、监控、模板等。
3. 高危权限单独拆分，不允许由“普通编辑权限”隐式继承。

### 验收标准
1. 越权访问返回明确的无权限提示。
2. 用户权限变更后重新登录或刷新 token 后生效。
3. 权限矩阵以《补充文档》为准。

## 8.3 实例管理

### 功能说明
1. 支持创建、查看、启动、停止、重启、删除实例。
2. 创建流程支持选择模板、资源规格、是否自动创建默认示例配置。
3. 实例删除采用软删除，进入保留期后可恢复或彻底清理。
4. 关键动作全部走异步 Job，并可查询进度。

### 验收标准
1. 创建实例后能生成独立运行绑定。
2. 删除中的实例不可再次启动或发布。
3. 同一实例上的互斥操作不可并发执行。

## 8.4 配置中心

### 功能说明
1. 支持结构化表单 + Raw JSON 双模式编辑。
2. 草稿可多次保存。
3. 发布前必须经过校验；失败时阻止发布。
4. 支持版本历史查看、差异对比、备注、回滚。
5. 敏感配置只允许后端解密注入运行时，前端只显示掩码。

### 验收标准
1. 未校验通过的版本不可发布。
2. 发布失败时当前 active 版本保持不变。
3. 回滚后自动生成新的 active 版本，而不是直接篡改历史版本。

## 8.5 模型 / Channel / Agent / Skill

### 功能说明
1. 平台侧负责配置与治理，不重做 OpenClaw 内部 channel 协议实现。
2. 支持模型提供商、channel、agent、skills 的基础管理。
3. 技能启用受白名单和来源审核约束。
4. Raw JSON 作为兜底能力，平台不承诺为所有长尾扩展做表单化适配。

### 验收标准
1. 技能未通过白名单时不可启用。
2. 配置页面能展示字段级校验错误。
3. 用户可查看某实例启用了哪些模型、channel、agent、skills。

## 8.6 节点中心

### 功能说明
1. 节点接入采用“配对申请 + 审批 + 绑定 + 在线状态同步”模型。
2. 节点状态拆分为：
   - 配对状态：pending / approved / rejected / expired
   - 在线状态：online / offline / error / detached
3. 一个节点同一时刻只能绑定一个实例。

### 验收标准
1. 未审批节点不可正式接入实例。
2. 节点长时间离线可触发告警。
3. 删除实例前需校验节点解绑策略。

## 8.7 监控、告警与通知

### 8.7.1 监控范围
1. 实例运行状态
2. 健康检查状态
3. channel probe 状态
4. model 状态
5. 节点在线状态
6. 使用量（请求数、活跃会话数、token / 成本估算）
7. 最近错误与诊断摘要

### 8.7.2 告警场景（V1）
1. 实例创建失败
2. 健康检查失败
3. channel probe 失败
4. 节点长时间离线
5. 凭证即将过期
6. 配置发布失败
7. 实例频繁重启

### 8.7.3 告警级别
- P1：严重（实例不可用 / 安全风险）
- P2：高（关键功能异常）
- P3：中（可降级运行）
- P4：低（提醒类）

### 8.7.4 通知机制（V1）
1. 站内通知：必做
2. 邮件通知：建议做
3. 企业 IM Webhook：预留到 V1.5
4. 通知需支持防骚扰节流（相同规则同实例 30 分钟内合并）
5. 用户可设置是否接收自己实例的邮件通知

### 验收标准
1. 告警必须可查看、确认、关闭，并保留审计。
2. 相同告警在抖动场景下不可无限刷屏。
3. 通知失败不影响实例运行，但要记录通知失败原因。

## 8.8 审计中心

### 功能说明
以下动作必须落审计：
- 登录 / 登出（成功与失败）
- 创建 / 启停 / 删除实例
- 保存草稿、校验、发布、回滚
- 节点审批 / 拒绝 / 解绑
- 技能启用 / 停用
- 用户角色变更
- 资源规格、版本 allowlist、模板策略变更
- 告警确认 / 关闭
- 密钥更新

### 验收标准
1. 敏感字段必须脱敏。
2. 审计日志支持按时间、操作人、实例、动作、结果筛选。
3. 审计服务异常时需有降级与补偿机制。

## 8.9 模板能力

### V1 范围
1. 创建实例时可选择：
   - 空白模板
   - 通用办公助手模板
   - 研发助手模板
   - 销售助手模板
2. 模板内容包括默认模型、默认 agent、默认 skills、默认安全策略、默认资源规格。

### V1.5 范围
1. 模板后台管理
2. 模板审核、上下架、版本控制
3. 模板权限与租户级可见范围控制

### 验收标准
1. V1 用户可在创建实例时选择模板。
2. 选择模板后仍可继续个性化修改。
3. 管理模板的后台能力不纳入 V1 开发承诺。

## 9. 关键业务规则

### 9.1 隔离规则

1. 用户之间的 OpenClaw 实例必须隔离。
2. 至少实现独立配置路径、独立状态路径、独立 workspace、独立日志、独立 secret scope。
3. 推荐实例级容器 / Pod；若暂无法容器化，需达到实例级进程 + 目录 + 凭证作用域隔离。
4. 不允许实例复用同一 workspace。

### 9.2 RBAC 规则

1. 高危操作需单独权限：
   - instance.delete
   - config.publish
   - config.rollback
   - node.approve
   - template.manage
   - platform.settings.manage
2. 只读角色不可见编辑按钮。
3. 权限矩阵见《补充文档》。

### 9.3 资源规格定义（V1 默认值）

| 规格 | vCPU | 内存 | 系统盘 | 建议并发会话 | 备注 |
|---|---:|---:|---:|---:|---|
| S | 1 | 1 GiB | 10 GiB | 1~3 | 轻量试用、单人低并发 |
| M | 2 | 2 GiB | 20 GiB | 3~10 | 默认推荐规格 |
| L | 4 | 4 GiB | 40 GiB | 10~20 | 高并发或多节点场景 |

规则：
1. V1 默认不开放自定义规格给普通员工。
2. 平台管理员可调整规格与配额。
3. 实际上限以平台部署资源和 runtime allowlist 为准。

### 9.4 OpenClaw 版本兼容策略

1. V1 只支持**平台批准的稳定版本列车**，不承诺兼容所有社区版本。
2. 版本策略采用：
   - `approved_runtime_version`：当前允许新建实例的默认版本
   - `supported_runtime_versions[]`：允许继续运行和变更的版本集合
   - `blocked_runtime_versions[]`：因安全或兼容问题被禁止的版本
3. 初始策略：
   - 仅允许当前稳定大版本列车（例如 `2026.2.x stable train`）
   - 不允许普通用户自行指定任意 patch 版本
4. 平台负责 runtime 版本编排与升级窗口控制。
5. 当上游出现安全修复时，平台管理员可通过 allowlist 快速下架不安全版本。

### 9.5 容量规划（V1 试点假设）

| 指标 | 规划值 |
|---|---:|
| 租户数 | 1~5 |
| 注册用户数 | 200 |
| 月活用户数 | 100 |
| 实例总数 | 300 |
| 同时运行实例数 | 120 |
| 节点总数 | 500 |
| 单租户实例上限 | 200 |
| 峰值 API 请求 | 150 RPS |

> 以上为内部试点默认规划值，正式排期前需由产品、技术、运维共同确认。

### 9.6 删除与恢复规则

1. 实例删除采用软删除。
2. 默认保留期 7 天。
3. 软删除期间同名实例不可重复创建，除非恢复或彻底清理。
4. 实例彻底删除前，必须处理节点解绑、任务终止、密钥归档。

### 10.5 浏览器兼容

1. Chrome：最近 2 个大版本
2. Edge：最近 2 个大版本
3. 不承诺 IE
4. Firefox/Safari 不作为 V1 正式兼容目标

### 10.6 可访问性（a11y）

1. 核心流程满足 WCAG 2.1 AA 的基础要求：
   - 键盘可达
   - 可见焦点
   - 文本对比度
   - 表单错误可读
2. 非核心页面如报表图表允许在 V1 做部分降级。

### 10.7 国际化（i18n）

1. V1 仅提供简体中文界面。
2. 文案资源必须抽离，避免硬编码。
3. V1.5 起可增加英文。

### 10.8 错误处理 UX

1. 字段级错误：展示在对应字段旁，并可跳转定位。
2. 页面级错误：使用错误区块展示原因、建议动作、重试入口。
3. 后台任务错误：支持查看 job 日志摘要。
4. 不允许只返回“系统异常”而没有 requestId / errorCode。

### 10.9 高危操作确认

1. 删除实例：需输入 `DELETE {instance_name}`
2. 回滚配置：需确认目标版本 + 风险提示
3. 节点解绑：需二次确认
4. 关闭高危技能：需记录原因并审计

### 10.10 降级策略

| 依赖异常 | 平台降级策略 |
|---|---|
| 审计服务异常 | 核心操作可继续；事件写入 outbox 重试；若 outbox 堆积超阈值则阻塞高危操作 |
| 监控采集失败 | 不影响实例运行；监控页显示“数据可能过期” |
| 配置校验服务超时 | 普通用户不可跳过校验；仅 break-glass 权限可强制发布并落审计 |
| 密钥服务异常 | 禁止创建 / 启动 / 发布等需要解密注入的操作 |
| Redis 异常 | 阻止新异步任务入队；只保留只读能力；前端提示平台处于降级状态 |

## 13. 验收标准（整体）

### 13.1 功能验收
1. 一个租户下可创建多个实例。
2. 不同用户只能看到自己有权限的实例。
3. 实例可完成创建、配置、校验、发布、启动、停止、重启、删除。
4. 节点可完成申请、审批、查看状态、解绑。
5. 管理员可查看监控、告警、审计。
6. 历史配置版本可查看 diff 并回滚。
7. 技能启用受白名单控制。

### 13.2 安全验收
1. 用户 A 不得访问用户 B 实例的配置、密钥、日志、节点。
2. 租户 A 不得访问租户 B 数据。
3. 敏感信息不得明文回显。
4. 所有关键动作可审计、可检索。
5. 不允许直接安装未审核公共技能。

### 13.3 运维验收
1. 实例异常可被监控发现并产生告警。
2. 发布失败不影响当前 active 版本。
3. 节点长时间离线可见且可告警。
4. 平台提供基础故障定位信息。
5. 平台健康检查与指标端点可被运维系统接入。

## 14. 风险与应对

### 14.1 把 OpenClaw 当成“共享多用户单实例”
- 风险：隔离不足、权限治理复杂、数据串扰
- 应对：V1 采用实例级隔离，不默认共享 runtime

### 14.2 社区技能供应链安全
- 风险：恶意技能、社工脚本、越权执行
- 应对：默认关闭公共技能市场，仅开放企业白名单

### 14.3 配置复杂度过高
- 风险：用户不会配、配错后无法启动
- 应对：模板化 + schema 驱动 + 严格校验 + Raw JSON 兜底

### 14.4 上游 OpenClaw 演进太快
- 风险：字段变化、兼容性变化、安全补丁频繁
- 应对：版本 allowlist + schema 驱动 + Runtime Adapter 统一收口

### 14.5 监控与审计变成“事后补丁”
- 风险：上线后难以排障与追责
- 应对：监控、告警、审计作为 V1 P0 能力，而非后补

## 16. 一句话结论

龙虾乐园 V1 的本质不是“做一个配置页面”，而是：

**OpenClaw Runtime + 企业控制台 + 运维治理层 + 安全治理层**

只要把下面这条链路打通，V1 就具备明确价值：

**登录 -> 创建实例 -> 编辑配置 -> 校验 -> 发布 -> 启动 -> 监控 -> 告警 -> 审计**
